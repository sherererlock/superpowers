# session-start — 在每次会话开始时把技能体系注入 AI 的上下文

## 技能定位与核心目标

`session-start` 是一个 SessionStart 钩子，它在每次 AI 会话启动时自动执行，将 `using-superpowers` 技能的完整内容注入到 AI 的上下文中。它解决的是一个根本性的引导问题：AI 在会话开始时并不知道 Superpowers 体系的存在，更不知道应该主动查找和调用技能。这个钩子的职责就是在 AI 开口之前，先把"你有超能力，这是使用手册"这条信息送达。

它不是一个功能性技能，而是整个 Superpowers 体系的**启动引导器**——没有它，其他所有技能都可能因为 AI 不知道去查找而永远沉睡。

## 核心设计哲学

### 只注入入口技能，不注入全部技能

钩子只读取并注入 `using-superpowers` 这一个技能的内容，而不是把所有技能都塞进上下文。这个选择背后有清晰的工程逻辑：`using-superpowers` 本身就是一个元技能，它告诉 AI 如何发现和调用其他技能。一旦 AI 理解了这个机制，它就能按需加载其他技能，而不需要在会话开始时就把所有技能都加载进来。

这与上下文窗口是公共资源的原则完全一致：只注入必要的引导信息，让后续的技能调用按需发生，而不是在会话开始时就消耗大量上下文空间。

### 用 `<EXTREMELY_IMPORTANT>` 标签强化注入内容的权重

注入的上下文被包裹在 `<EXTREMELY_IMPORTANT>` 标签中，并以"You have superpowers."作为开场白。这种措辞选择不是随意的——它在语义上给注入内容赋予了最高优先级，确保 AI 在处理后续对话时不会忽略这段引导信息。

相比于普通的系统提示，这种通过钩子注入的额外上下文需要更强的语气标记来确保被重视，因为它是在会话已经建立之后才被插入的，而不是作为系统提示的一部分从一开始就存在。

## 关键机制细节

### 跨平台输出格式的条件分发

钩子脚本中最复杂的部分是最后的平台检测逻辑。不同的 AI 平台对钩子输出的 JSON 格式有不同的期望：

- **Cursor**：通过 `CURSOR_PLUGIN_ROOT` 环境变量识别，期望 `additional_context`（snake_case）
- **Claude Code**：通过 `CLAUDE_PLUGIN_ROOT` 识别（且不是 Copilot CLI），期望嵌套的 `hookSpecificOutput.additionalContext`
- **Copilot CLI 及其他**：期望顶层的 `additionalContext`（SDK 标准格式）

这个三路分发逻辑解决了一个实际的跨平台兼容性问题：Superpowers 体系需要在多个 AI 平台上运行，但各平台的钩子 API 并不统一。通过环境变量检测而非配置文件，钩子在运行时自动适配当前平台，不需要用户手动配置。

注释中还特别说明了一个边界情况：Claude Code 会同时读取 `additional_context` 和 `hookSpecificOutput` 两个字段，且不去重，所以必须只输出当前平台对应的字段，否则内容会被注入两次。

### JSON 转义的纯 Bash 实现

钩子需要把技能文件的 Markdown 内容嵌入 JSON 字符串，这意味着需要对反斜杠、双引号、换行符等特殊字符进行转义。脚本实现了一个纯 Bash 的 `escape_for_json` 函数，使用参数替换（`${s//old/new}`）而非逐字符循环。

注释中明确说明了这个选择的原因："每个 `${s//old/new}` 是单次 C 级别的字符串遍历——比逐字符循环快几个数量级。"这是一个性能敏感的场景：钩子在每次会话启动时都会执行，如果转义函数很慢，会直接影响用户感知到的启动延迟。

### `printf` 替代 heredoc 的规避策略

脚本使用 `printf` 而非 heredoc 来输出 JSON，注释中解释了原因：bash 5.3+ 存在一个 heredoc 挂起的 bug（issue #571）。这是一个典型的"为已知 bug 主动规避"的工程决策——不等待上游修复，而是在代码层面绕开问题。

### 遗留目录检测与迁移警告

钩子在执行时会检查 `~/.config/superpowers/skills` 目录是否存在。如果存在，说明用户还在使用旧版本的自定义技能存储位置，钩子会生成一条警告消息，要求 AI 在第一次回复时告知用户迁移到新路径。

这个机制的设计有两个值得注意的细节：其一，警告被包裹在 `<important-reminder>` 标签中，并要求 AI "在看到此消息后的第一次回复中必须告知用户"——这是一种强制性的用户通知机制，确保迁移警告不会被 AI 静默忽略；其二，警告消息中明确告知了消除警告的方法（删除旧目录），给用户提供了明确的行动路径。

### hooks.json 的触发条件设计

`hooks.json` 中的 matcher 字段值为 `"startup|clear|compact"`，这意味着钩子不只在会话启动时触发，在用户执行 `clear`（清除对话历史）或 `compact`（压缩上下文）操作后也会重新触发。

这个设计解决了一个重要的状态管理问题：当用户清除对话历史或压缩上下文时，之前注入的 `using-superpowers` 内容可能已经不在 AI 的活跃上下文中了。通过在这些操作后重新触发钩子，确保 AI 在任何"上下文重置"事件后都能重新获得技能体系的引导信息，而不是在清除历史后突然"忘记"自己有超能力。
