# 赏析：executing-plans（执行计划技能）

## 技能定位与核心目标

`executing-plans` 是 Superpowers 工作流体系中的**执行引擎**，专门用于在独立会话中将已写好的实现计划付诸实施。它的核心目标是：在有人类伙伴监督的前提下，让 AI 代理以可控、可审查的方式逐步完成开发任务，而不是一口气盲目执行到底。

这个技能是整个 Superpowers 三段式开发流水线的中间环节：
- **上游**：`writing-plans` 负责生成计划
- **本技能**：`executing-plans` 负责执行计划
- **下游**：`finishing-a-development-branch` 负责收尾验证

---

## 核心设计哲学

### 1. 计划先行，批判性审查

技能的第一步不是立即动手，而是**先读计划、再批判性审查**。这一设计刻意引入了一个"暂停点"——如果 AI 发现计划有疑问或缺陷，必须先与人类伙伴沟通，而不是带着疑问硬上。

这背后的哲学是：**执行错误的计划比不执行更糟糕**。在开始之前花时间确认，远比执行到一半发现方向错误要经济得多。

### 2. 明确的停止条件（Stop Conditions）

技能专门列出了"何时停下来求助"的清单，包括：遇到阻塞、依赖缺失、测试失败、指令不清晰等。这是对 AI 代理"过度自信"倾向的主动对抗——**宁可停下来问，也不要猜测后继续**。

这种设计承认了 AI 的局限性：在不确定的情况下，人类判断比 AI 推断更可靠。

### 3. 子代理优先（Subagent-First）

技能在开头就明确提示：如果平台支持子代理（如 Claude Code 或 Codex），应优先使用 `subagent-driven-development` 而非本技能。这说明 `executing-plans` 是一个**降级方案**——在没有子代理能力时的最佳替代，而非最优解。

这种坦诚的自我定位非常罕见，体现了设计者对工具适用边界的清醒认知。

### 4. 透明性声明（Announce at Start）

技能要求 AI 在开始时主动声明"我正在使用 executing-plans 技能"。这是一种**元认知透明**设计——让人类伙伴随时知道 AI 当前处于哪个工作模式，便于监督和干预。

---

## 关键机制细节

### Step 1：加载与审查计划

- 读取计划文件后，AI 需要**批判性地**审查，而非盲目接受
- 如有疑虑，必须先提出，等待人类确认后再创建 TodoWrite 并开始
- 这一步的"批判性审查"是整个技能的安全阀

### Step 2：逐任务执行

- 每个任务遵循：标记为进行中 → 按步骤执行 → 运行验证 → 标记为完成
- 计划被设计为"bite-sized steps"（小粒度步骤），降低单步出错的影响范围
- 验证步骤是强制的，不可跳过

### Step 3：完成开发

- 所有任务完成后，自动衔接 `finishing-a-development-branch` 技能
- 同样要求主动声明切换技能，保持透明性

### 回退机制

- 当人类伙伴更新了计划，或发现根本性方向问题时，允许**回到 Step 1 重新审查**
- 这是一个显式的循环结构，而非单向流水线

### 分支保护

- 明确禁止在 main/master 分支上直接开始实现，除非用户明确同意
- 配合 `using-git-worktrees` 技能，要求在隔离工作区中操作

---

## 亮点总结

| 亮点 | 说明 |
|------|------|
| 批判性审查前置 | 执行前先质疑计划，避免带着错误假设盲目推进 |
| 明确的停止条件 | 列出具体的停止场景，对抗 AI 过度自信倾向 |
| 子代理优先声明 | 坦诚说明本技能是降级方案，引导用户选择更优路径 |
| 透明性声明机制 | 开始/切换技能时主动告知，便于人类监督 |
| 显式回退路径 | 支持回到审查步骤，不强迫单向执行 |
| 分支保护意识 | 禁止在主分支直接操作，配合 git worktree 隔离 |
| 三段式流水线集成 | 与 writing-plans、finishing-a-development-branch 无缝衔接 |

---

## 总结性评价

`executing-plans` 是一个**以可控性为第一优先级**的执行技能。它没有追求最大化自动化，而是在每个关键节点都设置了人类介入的机会：审查计划时、遇到阻塞时、完成后收尾时。

这种设计哲学与当前 AI 代理的实际能力边界高度契合——AI 擅长按步骤执行，但在面对歧义、缺失信息或意外情况时，人类判断仍然不可替代。技能通过明确的停止条件和透明性声明，将这种人机协作关系制度化，而不是依赖 AI 的"自我判断"。

对于需要在多会话中推进复杂开发任务的团队来说，这个技能提供了一个**低风险、高可审查性**的执行框架，是 Superpowers 工作流体系中不可或缺的一环。
